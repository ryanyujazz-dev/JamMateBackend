# System Contracts V2 — Compact

Current version: `v2_3_9`.

## Cross-layer contracts

- Pattern is pitchless and style-owned.
- Anticipation is a pitchless rewrite after PatternPlan and before Expression.
- Expression owns duration / release / velocity / articulation / touch / pedal.
- Voicing owns vertical pitch material, functional grouping, density, disposition, texture, and voice-leading.
- Realization converts plans to NoteEvent; it does not choose new voicings.
- MIDI rendering owns performed timing and channel/instrument maps.

## Harmony

`core/harmony/chord_parser.py` owns chord-symbol normalization and slash-bass metadata. Slash bass is metadata only unless a role explicitly consumes it. `material.py` owns chord-tone material and degree pitch-class resolution. `scale_resolver.py` owns `ResolvedScalePolicy`. `harmonic_context.py` owns `FunctionalMotion` and is reused by method-lock scope adapters.

## BassFoundation

- `max_region_span=12` / `max_region_span` enforces one-octave chord-region bass lines.
- `target_nextR_note` is the concrete next-root target note used for continuity.
- Root echo uses logical grid `(0.5, 2.5)`, exact same current-root note, `root_anchor_note`, `root_echo_same_as_region_root`, `timing_intent`, `root_echo_swing_upbeat`, and render-level swing timing.
- Classic fill `CF_TWO_BAR_TONIC_01` is `scene-triggered` through `bass_foundation_fill_scene` / `classic_fill_scene`, guarded by `classic_fill_enabled`, `_try_generate_classic_fill`, `classic_fill_min_gap_regions`, and `max_region_span`.
- BassFoundation Musicality Audit contains `Region Decision Trace` and is generated by `build_bass_foundation_audit` / `format_bass_foundation_audit_report`.
- Medium Swing BassFoundation keeps conservative `root echo density`, `scale_near_nextR`, `old-engine lane weighting`, `ThreeBeatSkeleton`, `R-R-*`, `40 : 40 : 10`, `legacy random`, `candidate_preflight`, `old_engine_legal_skeleton_pool`, `repeated_root_exact`, `repeated_root_notes`, and `repeated_root_violations`.
- Do not reintroduce `_apply_register_gravity_lane_caps`, `high_zone_upper_lane_cap`, or `very_high_zone_upper_lane_cap`.

## Voicing

`simultaneous_onset` names ordinary simultaneous attacks. `block_chord` is reserved only for discussion of traditional block harmonization / locked_hands / traditional block / block-chord terminology.

VoicingTextureIntent / VoicingTextureState live in `core/voicing/texture_plan.py`; LLM intent resolves to state. Disposition Projection lives under `core/voicing/disposition/`; method lock controls progression continuity.

## V2 LeadSheet contract

- `jammate_leadsheet_v2` keeps human-readable `sections` and `written_form` at the score boundary.
- `core/leadsheet.validation` is the input-quality gate: unknown sections, empty sections/bars, invalid chord symbols, unsorted/duplicate/out-of-range beats, invalid durations, and malformed repeat endings must raise `LeadsheetValidationError` before timeline compilation.
- Chord starts are one-based bar-local beats; durations are compiler-derived from next chord onset or bar boundary.
- `written_form` may contain repeat bodies, first/second endings, tags, and final endings; these compile into flat written bars before runtime chorus repetition.
- Performance repetitions live in `GenerationRequest.choruses`, not in the leadsheet document.
- The compiler must output chord-region-first `HarmonicRegion` objects carrying section/phrase/bar/form metadata for style policies, endings, fills, anticipation, and voicing texture.

## Pedal policy boundary

Pedal is an Expression contract, not a Pattern or Voicing contract. Patterns expose pedal-relevant facts such as tail availability, density, anticipation eligibility, and harmonic-rhythm pressure. Expression chooses `none/light/sustain/lush`; MIDI realizer only materializes approved piano pedal intent as CC64 with re-pedal offset. Bossa and Medium Swing stay dry by default; Jazz Ballad may use light/sustain pedal with chord-change re-pedal.
